## 一、工作台

### 1.1 统计概览

**统计卡片**:
- 人脉总数（点击跳转人脉管理）
- 互动记录数（点击跳转互动记录）
- 待处理提醒（点击跳转提醒事项）
- 圈子分布（商务圈/人才圈/创业圈/社交圈）

### 1.2 任务管理

**我的任务** Tab:
- 显示分配给我的任务（未来7天内，未完成）
- 支持快速创建任务
- 支持编辑、删除、更新状态
- 显示任务优先级、日期、负责人
- 操作：开始、完成、编辑、删除
- 逾期任务红色标记，今天的任务橙色标记

**我指派的任务** Tab（仅 leader/sales_director/admin）:
- 显示我创建并指派给他人的任务
- 查看任务执行状态
- 显示被指派人信息

**待跟进任务** Tab:
- 显示商机相关的待跟进任务
- 快速查看任务详情
- 点击可跳转到详细页面

**商机任务** Tab:
- 显示我负责的商机
- 筛选进行中和待处理的商机
- 显示商机状态和负责人

### 1.3 近期提醒

- 显示未完成的提醒事项（7天内）
- 按紧急程度标记：
  - 逾期：红色
  - 今天：橙色
  - 3天内：蓝色
  - 更远：灰色

### 1.4 最近互动

- 显示最近5条互动记录
- 显示互动类型、人脉、日期、结果

## 二、目标与计划

### 2.1 目标管理

**功能特性**:
- 创建和管理团队目标
- 支持按部门、季度、状态筛选
- 目标进度跟踪（百分比）
- 目标状态：进行中/已完成/已延期

**目标信息**:
- 标题、描述
- 负责人
- 所属部门
- 所属季度（Q1/Q2/Q3/Q4）
- 截止日期
- 完成进度（0-100%）
- 状态

**权限控制**:
- member：只能查看和编辑自己负责的目标
- leader：可以查看和管理本组的目标
- sales_director：可以查看和管理辖区内的目标
- admin：可以查看和管理所有目标

### 2.2 周报管理

**功能特性**:
- 组长和总监默认需要写周报
- admin 可以指定普通成员写周报
- 支持按周、部门筛选
- 自动填充本周一到周日日期

**周报内容**:
- 本周完成工作
- 下周工作计划
- 风险与问题

**权限控制**:
- member：只能查看和编辑自己的周报
- leader：可以查看本组成员的周报
- sales_director：可以查看辖区内的周报
- admin：可以查看所有周报，可以指定谁需要写周报

## 三、业务流转

### 3.1 线索池

**功能特性**:
- 统一管理所有业务线索
- 支持按状态、负责人、优先级、来源类型筛选
- 线索状态流转：新线索 → 跟进中 → 已转化/已关闭

**线索信息**:
- 线索标题
- 来源、来源类型（inbound/outbound/referral/event/other）
- 联系人、公司、联系方式
- 描述
- 状态（新线索/跟进中/已转化/已关闭）
- 负责人
- 优先级（高/中/低）

**关联功能**:
- 可以从线索创建策略
- 查看线索关联的策略和研发任务

**权限控制**:
- member：只能查看和编辑自己创建或负责的线索
- leader：可以查看和管理本组的线索
- sales_director：可以查看和管理辖区内的线索
- admin：可以查看和管理所有线索

### 3.2 策略管理

**功能特性**:
- 三维度策略管理：变现策略/流量策略/链路策略
- 支持按维度、岗位类型、状态筛选
- 可以关联来源线索

**策略信息**:
- 策略标题
- 维度（变现策略/流量策略/链路策略）
- 岗位类型（预算交付岗/流量运营岗）
- 描述
- 负责人
- 状态（草稿/进行中/已完成/已暂停）
- 来源线索（可选）

**关联功能**:
- 可以从策略创建研发任务
- 查看策略关联的研发任务
- 查看策略的来源线索

**权限控制**:
- member：只能查看和编辑自己负责的策略
- leader：可以查看和管理本组的策略
- sales_director：可以查看和管理辖区内的策略
- admin：可以查看和管理所有策略

### 3.3 研发任务

**功能特性**:
- 管理研发需求和任务
- 支持按状态、负责人、优先级、来源类型筛选
- 可以关联线索或策略
- 工时跟踪（预估工时/实际工时）

**任务信息**:
- 任务标题、描述
- 来源类型（线索/策略）
- 来源（关联的线索或策略）
- 负责人
- 状态（待开始/进行中/测试中/已完成/阻塞）
- 优先级（高/中/低）
- 预估工时、实际工时
- 开始日期、截止日期、完成日期

**统计功能**:
- 总任务数
- 待开始任务数
- 进行中任务数
- 已完成任务数

**权限控制**:
- member：只能查看和编辑自己创建或负责的任务
- leader：可以查看和管理本组的任务
- sales_director：可以查看和管理辖区内的任务
- admin：可以查看和管理所有任务

**业务流转链路**:
```
线索 → 策略 → 研发任务
```

## 四、商务协作

### 4.1 人脉管理

**功能特性**:
- 四大圈子分类：商务圈、人才圈、创业圈、社交圈
- 关系类型（可多选）：潜在客户、合作客户、外部人才、内部人才、创业伙伴、投资人/顾问、家人亲戚、朋友
- 动态表单：根据圈子和关系类型显示不同字段
- 人脉指派功能（组长/总监/admin 可用）

**商务圈字段**:
- 行业、关系等级、客户状态

**外部人才字段**:
- 现任公司/职位
- 目标职位
- 技能
- 潜力评级（S/A/B/C）
- 转化阶段（初步接触/深度沟通/意向明确/Offer阶段/已入职/已放弃）
- 意向程度（强/中/弱）
- 来源渠道
- 期望薪资

**内部人才字段**:
- 部门/职位
- 入职时间
- 技能
- 心脑口手四维评估

**通用字段**:
- 资源与诉求
- 权重（高/中/低）

**筛选功能**:
- 圈子分类
- 关系类型
- 城市
- 权重
- 搜索（姓名/公司/技能/标签）
- 人才圈专属筛选：潜力评级、转化阶段、意向程度

**批量导入**:
- 支持 CSV 批量导入
- 提供模板下载
- 支持中英文列名

### 4.2 互动记录

**功能特性**:
- 记录与人脉的所有互动
- 支持关联商机信息
- 自动创建提醒和待跟进任务

**互动类型**:
- 拜访、通话、送礼、餐饮、微信、邮件、会议、其他

**互动信息**:
- 互动类型
- 互动日期
- 关联人脉
- 信息重要程度（重要/中等/一般）
- 金额
- 结果/收获
- 下次跟进事项及日期

**商机信息**（可选）:
- 商机标题
- 商机状态
- 指派人
- 说明

**自动化功能**:
- 设置下次跟进日期后，自动创建提前3天的提醒
- 填写商机信息后，自动创建待跟进任务

**筛选功能**:
- 互动类型
- 城市
- 人脉权重
- 信息重要程度
- 日期范围（今天/昨天/最近7天/最近30天/本月/自定义）

### 4.3 提醒事项

**功能特性**:
- 管理所有待办提醒
- 支持手动添加和自动创建
- 一键标记完成

**提醒类型**:
- 跟进、生日、送礼、会议、其他

**状态显示**:
- 逾期（红色）
- 今天（橙色）
- 3天内（蓝色）
- 更远（灰色）

**筛选功能**:
- 显示全部 / 仅显示未完成

### 4.4 公司研究

**功能特性**:
- 管理竞品公司、同行异业公司、客户公司
- 支持多主体结构（集团总览 + 各主体 Tab）
- 研究摘要自动汇总（近30天）

**公司信息**:
- 公司分类（竞品/同行异业/客户）
- 行业、规模、成立年份
- 总部城市、官网
- 主营业务、商业模式
- 营收规模、标签

**集团总览**:
- 研究摘要卡片（近30天自动汇总）
  - 在册骨干数、运营产品数、在研产品数、30天动向数
  - 近30天人才变动（新入职/离职）
  - 最新5条动向快览
- 基本信息 Tab
- 全员架构 Tab
- 全部产品 Tab
- 动向记录 Tab
- 管理主体 Tab（增删改主体）

**各主体 Tab**:
- 主体基本信息（名称、注册名、城市、业务方向）
- 人员架构（仅显示该主体人员）
- 产品矩阵（仅显示该主体产品）

#### 4.4.1 人员架构

**功能特性**:
- 支持列表视图 / 组织架构图视图切换
- 支持设置直属上级，架构图自动生成树形结构
- 节点支持一键「加入人脉库」（转为外部人才）

**人员信息**:
- 姓名
- 级别（C级/VP/中层/高级骨干）
- 部门、职位
- 状态（在职/离职）
- 入职/离职时间
- 技能、背景

**架构图特性**:
- 节点颜色按级别区分
- 离职人员半透明显示

#### 4.4.2 产品矩阵

**功能特性**:
- 卡片式展示
- 产品状态跟踪

**产品信息**:
- 产品名称
- 产品类型
- 状态（运营中/内测中/开发中/已停止）
- 上线时间
- 描述
- 目标用户
- 核心功能

#### 4.4.3 动向记录

**功能特性**:
- 时间线展示
- 重要动向红色标注

**动向类型**:
- 人才动向、产品动向、业务动向、其他

**动向信息**:
- 标题、日期
- 重要程度
- 详细内容
- 信息来源
- 影响分析

#### 4.4.4 公司主体管理

**功能特性**:
- 支持在公司下创建多个法律主体/品牌
- 人员和产品可归属到具体主体
- 删除主体不删除人员/产品，仅解除归属关系

### 4.5 商机管理

**功能特性**:
- 从互动记录中提取的商机
- 支持按状态、负责人筛选
- 展开查看详细信息

**商机信息**:
- 商机标题
- 关联人脉
- 商机状态（待跟进/跟进中/已成交/已失败）
- 指派人
- 说明
- 创建时间

### 4.6 待跟进任务

**功能特性**:
- 从商机自动生成的待跟进任务
- 支持按状态筛选（待处理/进行中/已完成）
- 开始跟进、完成操作

**任务信息**:
- 任务标题
- 关联人脉
- 状态
- 创建时间
- 完成时间
- 完成备注

## 五、团队管理

### 5.1 任务看板

**功能特性**（仅 leader/sales_director/admin）:
- 按小组/成员分组查看任务
- 统计卡片：总任务数、待处理、进行中、已完成
- 支持按日期、状态筛选

**看板视图**:
- 按成员分组
- 显示每个成员的任务列表
- 显示任务状态、优先级、日期

### 5.2 送礼管理

#### 5.2.1 礼品库（admin 可见）

**功能特性**:
- 管理可送礼品
- 库存预警

**礼品信息**:
- 名称、分类
- 单价、库存、单位
- 描述

**库存预警**:
- ≤5：红色
- ≤20：橙色
- 充足：绿色

#### 5.2.2 送礼计划

**功能特性**:
- 创建节日/场合送礼计划
- 计划状态：草稿、进行中、已完成
- 在计划内查看所有申请进度

**计划类型**:
- 元旦、春节、中秋、生日、日常维护等

#### 5.2.3 审核与记录

**送礼申请流程**:
1. 成员在计划内选择人脉 + 礼品
2. 提交申请（可撤回）
3. 商务组长审核（通过/拒绝，填写审核意见）
4. 通过后自动扣减库存，生成送礼记录

**权限说明**:
- 成员：只能看自己的申请
- 组长：可看到所属成员的所有申请
- admin：可看到全部申请

**送礼记录**:
- 状态流转：待送出 → 已送出 → 已接收 / 已拒绝
- 送礼人可回填：
  - 实际送出日期
  - 收礼人满意度评分（1-5星）
  - 反馈内容
- 状态变为「已接收」时，自动在互动记录中生成一条「送礼」类型的记录

### 5.3 出差管理

#### 5.3.1 出差申请

**功能特性**:
- 创建出差申请
- 审批流程
- 费用录入

**申请信息**:
- 出差目的地
- 开始日期、结束日期

- 预估费用

**状态流转**:
- 草稿 → 待审批 → 已批准 / 已拒绝 → 已完成

**审批权限**:
- leader/sales_director/admin 可以审批

#### 5.3.2 费用明细

**功能特性**:
- 出差审批通过后才能录入费用
- 支持多种费用类型

**费用类型**:
- 交通、住宿、餐饮、礼品、其他

**费用信息**:
- 费用类型
- 日期
- 金额
- 描述

#### 5.3.3 报销单

**功能特性**:
- 出差完成后提交报销
- 审批流程

**报销流程**:
1. 出差完成后创建报销单
2. 提交审批
3. leader/admin 审批
4. 审批通过后状态变为「已支付」

#### 5.3.4 费用统计

**功能特性**:
- 月度费用趋势（近12个月）
- 按小组统计
- 按人员统计
- 按目的地统计

**筛选功能**:
- 年份、月份
- 小组

## 七、权限说明

### 7.1 角色权限

| 角色 | 权限说明 |
|------|---------|
| **admin**（超管） | 所有权限，管理用户/小组，查看全部数据 |
| **sales_director**（商务总监） | 查看自己及所辖所有小组（含下属组长带组）的数据 |
| **leader**（组长） | 查看自己及本组所有成员的数据，审批申请 |
| **member**（普通成员） | 查看/编辑自己负责的数据，发起送礼/出差申请 |
| **readonly**（只读） | 查看自己负责的数据，不可编辑 |
| **guest**（按模块授权） | 精细控制每个模块的读/写权限 |

### 7.2 数据可见范围

**"自己负责的数据"定义**:
- 人脉：`created_by = 自己` 或 `assigned_to = 自己`
- 原录入人始终保留编辑权限（即使被指派给他人）

**数据过滤规则**:
- **member/readonly**: 只能查看自己创建或负责的数据
- **leader**: 可以查看本组所有成员的数据
- **sales_director**: 可以查看所辖所有小组的数据
- **admin**: 可以查看所有数据

### 7.3 人脉指派

**功能说明**:
- 组长/总监/admin 可以指派人脉给团队成员
- 被指派人获得编辑权限
- 原录入人保留编辑权限
- 可以清空指派（取消指派）

### 7.4 通知系统

**通知触发场景**:
- 线索分配给用户时
- 线索状态变更时
- 策略分配给用户时
- 策略状态变更时
- 研发任务分配/重新分配给用户时
- 研发任务完成时通知创建人

**通知功能**:
- 右上角铃铛图标显示未读数量
- 下拉查看通知列表
- 点击通知跳转到对应页面

## 八、技术说明

### 8.1 技术栈

**前端**:
- React 18
- Ant Design 5
- React Router 6
- dayjs

**后端**:
- Express.js 5
- better-sqlite3（SQLite）
- JWT 认证

**数据存储**:
- SQLite 数据库（server/data.db）
- 单文件存储，备份方便

### 8.2 启动方式

**开发环境**:
```bash
# 根目录执行
npm run dev
```

**生产环境**:
```bash
# 方式1：直接运行
node server/index.js

# 方式2：Docker 部署
docker build -t relation-manager .
docker run -d -p 3001:3001 \
  -v $(pwd)/server/data.db:/app/server/data.db \
  --restart=always \
  --name relation-manager \
  relation-manager
```

**关闭服务**:
```bash
lsof -ti:3000,3001 | xargs kill -9
```
