经营小组-增长中台 功能说明
==============================
更新日期：2026-04-13

一、人脉管理助手
----------------

1. 工作台（Dashboard）
   - 人脉总数、互动记录数、待处理提醒统计
   - 近期提醒列表（7天内，逾期红色标注）
   - 最近互动记录（最新5条）

2. 人脉管理
   - 支持四大圈子分类：商务圈、人才圈、创业圈、社交圈
   - 关系类型（可多选）：潜在客户、合作客户、外部人才、内部人才、创业伙伴、投资人/顾问、家人亲戚、朋友
   - 动态表单：根据圈子和关系类型显示不同字段
     · 商务圈：行业、关系等级、客户状态
     · 外部人才：现任公司/职位、目标职位、技能、潜力评级、转化阶段、意向程度、来源渠道、期望薪资
     · 内部人才：部门/职位、入职时间、技能，以及心/脑/口/手四维评估
   - 资源与诉求字段（所有人通用）
   - 权重字段（高/中/低，红/橙/灰色标签）
   - 筛选：圈子分类、关系类型、城市、权重、搜索（姓名/公司/技能/标签）
   - 人才圈专属筛选：潜力评级、转化阶段、意向程度
   - 表格动态列：有外部人才时自动显示潜力评级/转化阶段/意向程度列
   - 内部人才行可展开查看心脑口手评估
   - 点击姓名查看详情（含互动记录、提醒子Tab）
   - 操作列快速打开互动记录抽屉
   - CSV 批量导入功能（提供模板下载，支持中英文列名）

3. 互动记录
   - 互动类型：拜访、通话、送礼、餐饮、微信、邮件、会议、其他
   - 信息重要程度：重要、中等、一般
   - 金额、结果/收获、下次跟进事项及日期
   - 设置下次跟进日期后，自动创建提前3天的提醒
   - 筛选：互动类型、城市、人脉权重、信息重要程度、日期范围
     · 日期范围支持快捷选项：今天、昨天、最近7天、最近30天、本月、自定义区间
   - 送礼记录状态变为「已接收」时，自动生成一条类型为「送礼」的互动记录

4. 提醒事项
   - 支持类型：跟进、生日、送礼、会议、其他
   - 状态显示：逾期（红）、今天（橙）、3天内（蓝）、更远（灰）
   - 支持手动添加提醒
   - 一键标记完成
   - 显示全部 / 仅显示未完成切换


二、公司研究助手
----------------

1. 公司研究（列表页）
   - 公司分类：竞品公司、同行异业公司、客户公司
   - 基本信息：行业、规模、成立年份、总部城市、官网、主营业务、商业模式、营收规模、标签
   - 搜索、分类筛选

2. 公司详情（Drawer）
   支持多主体结构（集团总览 + 各主体 Tab）

   集团总览包含：
   · 研究摘要卡片（近30天自动汇总）
     - 在册骨干数、运营产品数、在研产品数、30天动向数
     - 近30天人才变动（新入职/离职）
     - 最新5条动向快览
   · 基本信息 Tab
   · 全员架构 Tab
   · 全部产品 Tab
   · 动向记录 Tab
   · 管理主体 Tab（增删改主体）

   各主体 Tab 包含：
   · 主体基本信息（名称、注册名、城市、业务方向）
   · 人员架构（仅显示该主体人员）
   · 产品矩阵（仅显示该主体产品）

3. 人员架构
   - 支持列表视图 / 组织架构图视图切换
   - 人员信息：姓名、级别（C级/VP/中层/高级骨干）、部门、职位、状态（在职/离职）、入职/离职时间、技能、背景
   - 支持设置直属上级，架构图自动生成树形结构
   - 架构图节点颜色按级别区分，离职人员半透明显示
   - 节点支持一键「加入人脉库」（转为外部人才）

4. 产品矩阵
   - 卡片式展示，包含产品名称、类型、状态、上线时间、描述、目标用户、核心功能
   - 产品状态：运营中、内测中、开发中、已停止

5. 动向记录
   - 动向类型：人才动向、产品动向、业务动向、其他
   - 时间线展示，重要动向红色标注
   - 字段：标题、日期、重要程度、详细内容、信息来源、影响分析

6. 公司主体管理
   - 支持在公司下创建多个法律主体/品牌
   - 人员和产品可归属到具体主体
   - 删除主体不删除人员/产品，仅解除归属关系


三、送礼管理
------------

1. 礼品库（admin 可见）
   - 管理可送礼品：名称、分类、单价、库存、单位、描述
   - 库存颜色预警：≤5 红色、≤20 橙色、充足绿色
   - 审核通过后自动扣减库存

2. 送礼计划
   - 创建节日/场合送礼计划（元旦/春节/中秋/生日/日常维护等）
   - 计划状态：草稿、进行中、已完成
   - 在计划内查看所有申请进度

3. 送礼申请流程
   成员在计划内选择人脉 + 礼品 → 提交申请（可撤回）
   → 商务组长审核（通过/拒绝，填写审核意见）
   → 通过后自动扣减库存，生成送礼记录

   - 成员只能看自己的申请
   - 组长可看到所属成员的所有申请
   - admin 可看到全部申请

4. 送礼记录
   - 状态流转：待送出 → 已送出 → 已接收 / 已拒绝
   - 送礼人可回填：实际送出日期、收礼人满意度评分（1-5星）、反馈内容
   - 状态变为「已接收」时，自动在互动记录中生成一条「送礼」类型的记录


四、权限与用户管理
------------------

角色说明：
┌──────────────┬────────────────────────────────────────────┐
│ 角色         │ 权限说明                                   │
├──────────────┼────────────────────────────────────────────┤
│ 超管 admin   │ 所有权限，管理用户，查看全部数据           │
│ 商务组长     │ 审核所属组员送礼申请，查看组员数据         │
│ 普通成员     │ 增删改自己录入的数据，发起送礼申请         │
│ 只读访客     │ 查看所有数据，不可编辑                     │
│ 按模块授权   │ 精细控制每个模块（人脉/公司/互动/提醒）    │
│              │ 的读/写权限                                │
└──────────────┴────────────────────────────────────────────┘

用户管理：
- 创建用户时可设置归属组长（member 专属）
- 支持修改密码
- 默认管理员：admin / admin123（首次启动自动创建）


五、技术说明
------------
- 前端：React 18 + Ant Design 5
- 后端：Express 5 + better-sqlite3（SQLite）
- 认证：JWT（7天有效期）
- 数据存储：server/data.db（单文件，备份方便）
- 部署：Docker（Dockerfile 已配置，使用国内镜像源）
- 代码仓库：https://github.com/hausenchan/Relation

启动方式：
- 开发：npm run dev（根目录执行）
- 生产：docker build + docker run，或直接 node server/index.js
- 默认端口：后端 3001，前端开发 3000
