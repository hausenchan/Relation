# 线索策略需求链路展示 PRD

## 1. 背景

当前系统中已经存在线索、策略、需求三个业务对象，但在实际使用中，列表和详情页对这三者之间的关联展示还不够清晰，存在以下问题：

- 线索列表里无法直接看到当前线索的跟进结果
- 策略虽然可以关联来源线索，但列表里缺少更明确的线索标识
- 策略列表缺少策略 ID，跨页面沟通时不方便快速定位
- 需求列表中缺少“关联线索”和“关联策略”的直观字段

因此，需要补齐“线索 -> 策略 -> 需求”这条链路在页面上的标识和展示能力，让用户在列表层面就能快速识别上下游关系。

## 2. 目标

- 在线索列表中增加“跟进结果”字段，并支持编辑录入
- 在线索列表中增加“线索ID”字段，用于唯一识别线索
- 在策略列表中增加“策略ID”字段
- 在策略列表中，将“来源线索”优先展示为线索 ID
- 在策略详情中可查看来源线索标题
- 在需求列表中增加“关联线索”和“关联策略”字段
- 让用户在列表页就能快速识别对象之间的上下游关系

## 3. 适用用户

- 商务人员
- 产运人员
- 研发人员
- 管理层查看者

## 4. 范围

### 4.1 本次范围

- 线索列表字段增强
- 线索编辑字段增强
- 策略列表字段增强
- 策略详情里的来源线索展示增强
- 需求列表字段增强

### 4.2 不在本次范围

- 业务编号改造成带前缀格式（如 `L-0001`、`S-0001`）
- 自动生成链路图
- 多对多复杂追溯图谱
- 新增独立链路分析页面

## 5. 核心业务链路

本次需要强化展示的业务链路为：

**线索 -> 策略 -> 需求**

含义：

- 策略可以来源于某条线索
- 需求可以关联某条策略
- 用户应能从列表和详情中快速看出这条链路关系

## 6. 线索模块需求

### 6.1 列表新增字段

在线索列表中新增以下字段：

- 线索ID
- 跟进结果

### 6.2 字段定义

#### 线索ID

- 用于唯一标识一条线索
- 第一版默认直接使用系统现有主键 ID 展示
- 例如：`12`

#### 跟进结果

- 用于填写和展示该线索当前跟进后的结果摘要
- 示例：
  - 已与客户初步沟通，待确认预算
  - 客户暂无明确意向
  - 已转化为策略跟进

### 6.3 编辑能力

在线索编辑页面/弹窗中新增字段：

- 跟进结果

要求：

- 支持新建时填写
- 支持编辑时更新
- 列表中展示最新保存值

### 6.4 线索列表目标

用户应能在列表中快速回答：

- 这条线索的唯一 ID 是什么？
- 当前跟进结果怎么样？

## 7. 策略模块需求

### 7.1 策略列表新增字段

在策略列表中新增：

- 策略ID

### 7.2 来源线索展示规则

策略列表中已有“来源线索”字段，本次要求改为：

- 优先显示来源线索的线索ID
- 而不是只显示线索标题

示例：

- 来源线索：`12`

### 7.3 策略详情展示规则

策略详情页中的“来源线索”展示要求：

- 列表里看线索ID
- 详情里可看到线索标题

也就是说：

- 列表层用于快速定位
- 详情层用于具体识别线索内容

### 7.4 策略列表目标

用户应能在策略列表中快速回答：

- 这条策略的策略ID是什么？
- 它来源于哪条线索？

## 8. 需求模块需求

### 8.1 列表新增字段

在需求列表中新增以下字段：

- 关联线索
- 关联策略

### 8.2 字段定义

#### 关联线索

- 显示该需求关联到的线索标识
- 第一版建议优先显示线索ID
- 无关联时显示 `-`

#### 关联策略

- 显示该需求关联到的策略标识
- 第一版建议优先显示策略ID
- 无关联时显示 `-`

### 8.3 展示目标

用户应能在需求列表中快速回答：

- 这条需求在跟哪条线索有关？
- 这条需求是在推进哪条策略？

## 9. 展示口径统一规则

为了统一跨模块理解，列表与详情页的展示口径定义如下：

### 9.1 列表页

- 列表页优先展示 ID
- 用于快速定位和跨页面沟通

具体规则：

- 线索列表展示线索ID
- 策略列表展示策略ID
- 策略列表中的来源线索显示线索ID
- 需求列表中的关联线索显示线索ID
- 需求列表中的关联策略显示策略ID

### 9.2 详情页

- 详情页优先展示标题/名称
- 用于理解业务语义

具体规则：

- 策略详情中可看到来源线索标题
- 后续如有需要，需求详情中也可进一步展示关联线索标题和策略标题

## 10. 数据关系复用

本次优先复用现有字段关系，不新增复杂关联表。

### 10.1 策略来源线索

复用：

- `strategies.source_type`
- `strategies.source_id`

规则：

- 当 `source_type = 'lead'`
- `source_id = leads.id`

表示该策略来源于该线索。

### 10.2 需求关联策略

复用：

- `dev_tasks.source_type`
- `dev_tasks.source_id`

规则：

- 当 `source_type = 'strategy'`
- `source_id = strategies.id`

表示该需求关联该策略。

### 10.3 需求关联线索

若当前需求列表需要直接显示“关联线索”，可采用以下优先级：

1. 若需求本身直接来源于线索，则显示该线索
2. 若需求来源于策略，且策略来源于线索，则回溯显示该来源线索
3. 若无法识别，则显示 `-`

## 11. 第一版实现建议

### 11.1 线索模块

- 在线索表中增加 `follow_result` 字段
- 列表新增“线索ID”“跟进结果”
- 编辑页新增“跟进结果”

### 11.2 策略模块

- 列表新增“策略ID”
- “来源线索”列改为显示线索ID
- 详情中补充来源线索标题

### 11.3 需求模块

- 列表新增“关联线索”“关联策略”
- 优先展示 ID

## 12. 验收标准

### 12.1 线索模块

- 线索列表可看到“线索ID”
- 线索列表可看到“跟进结果”
- 编辑线索时可填写和修改“跟进结果”
- 保存后列表可正确展示

### 12.2 策略模块

- 策略列表可看到“策略ID”
- 策略列表中的“来源线索”显示线索ID
- 点击策略详情可看到来源线索标题

### 12.3 需求模块

- 需求列表可看到“关联线索”
- 需求列表可看到“关联策略”
- 无关联数据时显示 `-`

### 12.4 链路识别

- 用户可从列表页快速看出：
  - 某条线索的 ID 和当前跟进结果
  - 某条策略来源于哪条线索
  - 某条需求关联哪条线索和哪条策略
