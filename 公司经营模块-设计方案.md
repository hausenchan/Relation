# 公司经营模块 - 需求分析与设计方案

**设计日期**: 2026-04-14  
**目标用户**: CEO、COO、CTO、CMO（高管层）

---

## 一、需求概述

### 1.1 业务背景

高管层需要一个专属的"公司经营"视角，用于：
- 快速了解公司核心资源（高级人才、重点客户）
- 实时掌握竞争态势（竞品公司动态）
- 定期进行战略决策和经营复盘（战略月会、经营周会）

### 1.2 核心需求

| 模块 | 功能 | 数据来源 |
|------|------|---------|
| 高级人才 | 展示重要的外部人才 | 人脉管理（外部人才 + 高权重） |
| 竞争公司动态 | 展示竞品公司最新动向 | 公司研究（竞品公司动向） |
| 重点客户维护 | 展示重要客户及最近联系记录 | 人脉管理（合作客户 + 高权重） + 互动记录 |
| 经营周报 | 战略月会 + 经营周会记录 | 新建表（executive_reports） |

### 1.3 权限要求

- **严格权限控制**: 仅 CEO、COO、CTO、CMO 可访问
- **协同编辑**: 四位高管共同编辑同一份会议记录
- **数据隔离**: 其他角色（包括 admin）不可见

## 二、数据库设计

### 2.1 用户角色扩展

**方案**: 在 `users` 表中新增 `executive_role` 字段

```sql
-- users 表新增字段
ALTER TABLE users ADD COLUMN executive_role TEXT DEFAULT NULL;
-- 可选值: 'ceo', 'coo', 'cto', 'cmo'
```

**说明**:
- `executive_role` 为 NULL 表示非高管
- 只有设置了 `executive_role` 的用户才能访问"公司经营"模块
- 一个用户可以同时拥有 `role`（如 admin）和 `executive_role`（如 ceo）

### 2.2 经营周报表

```sql
CREATE TABLE IF NOT EXISTS executive_reports (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  
  -- 会议类型
  report_type TEXT NOT NULL, -- 'strategic_monthly' | 'operational_weekly'
  
  -- 会议时间
  meeting_date TEXT NOT NULL, -- 会议日期 YYYY-MM-DD
  year INTEGER NOT NULL,
  month INTEGER NOT NULL,
  week INTEGER, -- 经营周会才有，第几周（1-4）
  
  -- 简报部分（经营周会）
  weekly_results TEXT, -- 本周核心结果
  key_judgment TEXT, -- 最重要的判断
  decision_needed TEXT, -- 需要决策的问题
  next_week_actions TEXT, -- 下周建议动作
  
  -- 决策部分（经营周会）
  key_issues TEXT, -- 核心问题（2-3个）
  decisions TEXT, -- 决策结果
  
  -- 战略月会内容
  strategic_direction TEXT, -- 公司方向
  key_focus TEXT, -- 重点做什么
  monthly_summary TEXT, -- 月度总结
  
  -- 参会人员（JSON 数组，存储 user_id）
  attendees TEXT, -- ["1", "2", "3", "4"]
  
  -- 编辑记录
  last_edited_by INTEGER, -- 最后编辑人
  last_edited_at DATETIME,
  
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_executive_reports_type_date ON executive_reports(report_type, meeting_date);
CREATE INDEX idx_executive_reports_year_month ON executive_reports(year, month);
```

### 2.3 数据视图设计

#### 2.3.1 高级人才视图

```sql
-- 从 persons 表筛选
SELECT * FROM persons 
WHERE person_category = 'talent' 
  AND relation_types LIKE '%talent_external%'
  AND importance IN ('high', 'vip')
ORDER BY updated_at DESC;
```

#### 2.3.2 竞争公司动态视图

```sql
-- 从 company_dynamics 表筛选
SELECT 
  cd.*,
  c.name as company_name,
  c.industry
FROM company_dynamics cd
LEFT JOIN companies c ON cd.company_id = c.id
WHERE c.category = 'competitor'
  AND cd.importance = 'high'
ORDER BY cd.date DESC
LIMIT 20;
```

#### 2.3.3 重点客户维护视图

```sql
-- 从 persons 表筛选，关联最近互动
SELECT 
  p.*,
  i.date as last_interaction_date,
  i.type as last_interaction_type,
  i.result as last_interaction_result,
  CAST(julianday('now') - julianday(i.date) AS INTEGER) as days_since_last_contact
FROM persons p
LEFT JOIN (
  SELECT person_id, MAX(date) as max_date
  FROM interactions
  GROUP BY person_id
) latest ON p.id = latest.person_id
LEFT JOIN interactions i ON p.id = i.person_id AND i.date = latest.max_date
WHERE p.person_category = 'business'
  AND p.relation_types LIKE '%customer_active%'
  AND p.importance IN ('high', 'vip')
ORDER BY days_since_last_contact DESC;
```

```javascript
// 高管权限检查中间件
function executiveOnly(req, res, next) {
  const { executive_role } = req.user;
  if (!executive_role || !['ceo', 'coo', 'cto', 'cmo'].includes(executive_role)) {
    return res.status(403).json({ error: '仅高管可访问' });
  }
  next();
}
```

## 六、注意事项

### 6.1 权限安全

1. **严格的权限控制**
   - 后端 API 必须使用 `executiveOnly` 中间件
   - 前端路由也要检查权限
   - 菜单仅在 `user.executive_role` 存在时显示

2. **数据隔离**
   - 高管数据与其他模块完全隔离
   - 即使是 admin 也不能访问（除非同时是高管）

### 6.2 协同编辑

1. **编辑冲突处理**
   - 显示"最后编辑人"和"最后编辑时间"
   - 提示用户可能存在的编辑冲突
   - 建议：实时协同编辑（可选，使用 WebSocket）

2. **版本历史**（可选）
   - 保存每次编辑的历史版本
   - 支持查看历史和回滚

### 6.3 会议提醒

1. **自动提醒**
   - 战略月会：每月最后一个周末前3天提醒
   - 经营周会：每周五提醒（每月前3周）

2. **提醒方式**
   - 系统通知
   - 邮件提醒（可选）

### 6.4 数据展示

1. **高级人才**
